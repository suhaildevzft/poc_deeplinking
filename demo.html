üìã How It Works

1. HTML Meta Tags

The main index.html includes these important meta tags:

<!-- iOS Deep Link Meta Tags -->
<meta name="apple-itunes-app" content="app-id=6739358459">
<meta property="al:ios:app_store_id" content="6739358459">
<meta property="al:ios:app_name" content="Deep Link POC">
<meta property="al:ios:url" content="deeplinkpoc://example.com/dashboard">

<!-- Android Deep Link Meta Tags -->
<meta property="al:android:package" content="app.slavic.customer.uat">
<meta property="al:android:app_name" content="Deep Link POC">
<meta property="al:android:url" content="deeplinkpoc://example.com/dashboard">

2. App Detection Logic

The JavaScript code uses multiple methods to detect if the app is installed:

  • Visibility Change: Detects when the page becomes hidden (app opens)
  • Page Blur: Detects when the page loses focus
  • Timeout Method: If app doesn't open within 3 seconds, assumes not installed
  • Custom URL Scheme: Attempts to open the app using custom scheme

3. Smart Fallbacks

If app is not detected:

  • iOS users are redirected to App Store
  • Android users are redirected to Play Store
  • Web users continue in browser

üì± Mobile App Configuration

Android (AndroidManifest.xml)

<intent-filter android:autoVerify="true">
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="deeplinkpoc" />
</intent-filter>

iOS (Info.plist)

<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleURLName</key>
        <string>deeplinkpoc.deeplink</string>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>deeplinkpoc</string>
        </array>
    </dict>
</array>
